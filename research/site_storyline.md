# Storyline будущего сайта

Дата ревизии: 2026-05-08.

Сайт пока не меняем. Этот файл фиксирует структуру будущей презентации, чтобы дизайн потом не был просто красивой оболочкой без аргумента.

## Референс по стилю

Референс `kawka.me` полезен не цветами, а ощущением:

- минималистичный, дорогой, спокойный интерфейс;
- много воздуха;
- сильная типографика;
- короткие формулировки;
- плавное движение и микроанимации;
- секции строятся как рассказ, а не как набор карточек.

Для Dream Queue AI это можно адаптировать в B2B-стиле: меньше “личного портфолио”, больше precision, цифр и операторского языка.

## Первый экран

Цель: за 5 секунд объяснить пилотную гипотезу и evidence posture.

Вариант headline:

> Пилотный слой управления временем гостя.

Supporting copy:

> Dream Queue AI предлагает проверить virtual queue slots, wait prediction, guest routing, risk controls и ограниченное priority inventory на выбранных bottleneck attractions.

Primary CTA:

> Посмотреть сценарий пилота

Secondary CTA:

> Открыть расчеты

## Структура страницы

1. Problem: physical waiting may reduce perceived value and available time in other zones.
2. Evidence: review metrics из seed-выборки с честной пометкой Source / limited sample.
3. Benchmark: Genting SkyWorlds + Alibaba Cloud как category proof, not KPI proof.
4. Product: virtual queue, forecasting, routing, risk controls, operator dashboard.
5. Simulation: baseline vs optimized as Model output, not measured park performance.
6. Pilot plan: 4-8 недель, ограниченный набор аттракционов, baseline first.
7. Ask: встреча / доступ к данным / paid pilot.

Контентная база для страницы: `research/ostrov_mechty/case_study.md` и `research/product_application.md`.

## Визуальные блоки

- “Before / after” поток гостей.
- Мини-карта парка с bottlenecks.
- График queue variance.
- Revenue calculator.
- Operator dashboard preview.
- Guest journey: билет -> слот -> маршрут -> аттракцион -> еда/retail.

## Тон

Нужно звучать как операционный продукт для парка, а не как студенческий AI-demo.

Правильные слова:

- capacity;
- guest flow;
- bottlenecks;
- wait-time forecast;
- slot inventory;
- scenario revenue;
- pilot;
- assumptions;
- evidence.

Слова, которые лучше не ставить в центр:

- “магически убираем очереди”;
- “точно увеличим прибыль на X%”;
- “заменим приложение парка”;
- “полностью автоматизируем парк”.
