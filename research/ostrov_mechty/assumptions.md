# Assumptions

Assumption - это рабочая гипотеза модели, а не факт из источника.

## Данные

- Отзывы теперь не синтетические: это real-sourced seed dataset с URL.
- Выборка отзывов пока маленькая и не репрезентативная.
- Список аттракционов взят с официальных страниц, но пока неполный.
- Реальных данных о посещаемости, capacity и фактическом ожидании у нас нет.
- Benchmark-датасеты можно использовать только для сравнения методологии и порядков величин.

## Модель

- `estimated_capacity_per_hour` - оценка для симуляции, не официальный показатель.
- `estimated_popularity` - экспертная MVP-оценка, не данные парка.
- `queue_risk_score` - служебный индекс для ранжирования, а не “истина”.
- Симуляция нужна для pitch-визуала и обсуждения гипотезы, а не для точного прогноза.
- `visitor_satisfaction_score` / `satisfaction_score` - условный индекс, а не измеренная NPS/CSAT-метрика.

## Операционные ограничения

- Нельзя увеличить физическую пропускную способность аттракциона “магией”.
- Продукт может только перераспределять спрос, сглаживать пики и управлять слотами.
- Платные fast slots должны занимать ограниченную долю capacity, иначе будет социальный негатив.
- Бесплатные виртуальные слоты нужны, чтобы продукт не выглядел как чистый paywall.

## Бизнес-модель

- Текущие revenue-share параметры в `configs/assumptions.yaml` - рабочая гипотеза.
- Их нельзя выдавать за договоренность с парком.
- Финмодель надо использовать как сценарный калькулятор.
